CTS2 in LexEVS

The Javascript Widget Site

LexEVS is an open source terminology service implementation. As such it provides much of the functionality of the specification and standard known as CTS2, or the Common Terminology Service created in conjunction with data standards groups Health Level 7 and the Object Management Group. LexEVS 6.1 features an implementation of the CTS2 standard as a REST Service. Click on the tabs above to see a variety of examples written in Javascript.

LexEVS

LexEVS supports the Enterprise Vocabulary Service hosted by the Center for Biomedical Informatics and Information Technology, or CBIIT at the National Cancer Institute. Much of the documentation for LexEVS as well as application downloads are found here. Documentation of the CTS2 compatible API that runs on top of LexEVS is found here.

Examples

The examples are based on the popular Bootstrap JavaScript framework along with its Typeahead plugin. Use of this platform allows some fairly concise examples of JavaScript implementations of REST clients that run against the CTS2 implementation running on top of LexEVS. Finally, because REST calls are by nature handled as cross site calls we sometimes use a JQuery extension called jsonp. We recommend you use the versions available in the lexevscts2 github repository to insure compatibility.

CTS2

The Common Terminology Services 2 API specification is hosted by OMG at this location. HL7's Normative Service Functional Model is described here. Helpful reference implementation information and a summary of functionality is hosted by the Mayo Clinic as CTS2 reference implementations.
